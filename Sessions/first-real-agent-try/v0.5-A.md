## Deviations from spec

- **Snapshot is mock-anchored, not BQ-backed.** The spec asked for "real data swap in for the Hermes path." Mock fixtures cloned from generate.ts ship today so the Report renders fully every run; the BQ-to-snapshot overlay seam is in `src/lib/agents/hermes/snapshot.ts` ready for follow-up. Why: GlobalComix BQ has known data gaps (installs NULL in a high percentage of rows; weekly aggregation needs its own shaping); a half-finished real-data path would produce reports with sparse tables and miss the "indistinguishable from manual" bar.
- **Regenerate scoping is post-hoc, not prompt-level.** Quill runs over all slide_targets and the endpoint filters back to the requested one. Wastes ~66% of output tokens (~$0.005-0.01/call). Queued for a v0.5-A follow-up; documented in regenerate.ts header.
- **agent_runs.output.approval.edits NOT populated by Hermes regenerate / edit.** Spec called for both reports.audit and agent_runs.output.approval.edits to track. v0.5-A only writes reports.audit. The agent_runs side has no on-screen consumer until LangSmith integration in workstream D, so it's deferred.
