**Reviewer should-fix:**
- Skills frontmatter parser drops multiline `description:` values silently (regex is single-line). All three current skills are single-line so works today; tighten when a folded/multiline skill lands.
- extractSenderEmail length-tie sort is engine-defined for equal-length addresses; add an alpha tiebreak for reproducibility.
- "Gmail not connected" pill is hardcoded today; will become a real status when workstream C wires OAuth.

**Security should-fix carry-over to workstream C:**
- PREVIEW_USER_ID ("preview-user") is a stable string. Hard-gated to non-prod (LUMEN_PREVIEW + NODE_ENV check) so the practical risk is negligible, but namespacing it per-process would close the theoretical collision window.
- agent_memory_kv stores the first 280 chars of email_text per parse_intent call. TODO already flagged in parse-intent.ts:249-253 for redaction or TTL shortening before Gmail OAuth widens the input source.
- contacts.getContactByEmail uses ilike on the pre-normalised lowercase string; user-supplied `%` or `_` in a local-part (legal but rare) would match more rows than intended. Escape both chars or switch to `eq` on `lower(email)` using the existing functional index.

**Performance nits:**
- loadHermesSkills re-reads three SKILL.md files per page render. Wrap in React.cache + module-scoped Map for cold-start savings on Vercel. ~3ms today.
- listRecentHermesRunsForUser does a seq-scan via input->>'user_id'. Fine at n=11; add an expression index when row count grows.

**Accessibility should-fix:**
- "..." placeholders in tables are read literally by screen readers. Wrap as `<span aria-hidden>...</span>` + SR-only "Not set" / "No report" label.
- Skill slug under `text-muted` may drop below 4.5:1 on glass surface; bump to text-secondary or mark aria-hidden since the human-readable name is already exposed.
- The "In use" skill dot needs `role="img"` (not just aria-label) so generic-span SR/browser pairs announce it.
