What's in the repo right now

  • BQ client. Singleton, ADC + service-account JSON. src/lib/bq.ts
  • Security layer. Allowlist via env, per-client schema map, two query strategies. src/lib/bq-security.ts
  • Query helpers. KPIs, trend, channel mix, campaigns, freshness, data bounds, all parameterized and cached at 30 min. src/lib/bq-queries.ts
  • API routes. Full set under /api/bq/* plus per-client variants under /api/bq/100play/*
  • UI components. KpiCard, TrendChart, ChannelMix, CampaignsTable, ClientSelector, DataFreshnessBar — all built and rendering
  • 3 clients served: GlobalComix, Playw3, 100play (legacy strategies)
New in Phase 1 ~20 hours

What we're adding

  • Third query strategy: management-dashboard joins the 4-platform union
  • Roster integration. pre_sales_updated_clients_tracking drives the active client list with team, budget, account manager
  • UNION-ALL helper. One TS function emits the 4-table union for every query. src/lib/bq-union.ts
  • MD query module. Mirror of bq-queries.ts for the union case. src/lib/bq-queries-md.ts
  • Per-platform freshness. Direct MAX(date) from the 4 tables, not Rivery telemetry
  • Roster-driven picker. ClientSelector fetches /api/bq/roster and renders all 8 active UA clients
  • Apple unattributed footnote. Surfaces the NULL master_account percentage honestly
  • Service account. lumen-app identity so audit logs can distinguish Lumen from ETL

Risks and mitigations
What could break and how we handle it without external dependencies.
BI silently renames a column
Mitigation: Explicit column names in the union helper. Daily synthetic query (Vercel cron) hits every route, alerts on failure. Recovery is one-file change.
"new" dashboard layer takes over
Mitigation: Union helper is one file, one place. Migration is 30 minutes, not a rewrite. We'll see it in the synthetic monitor first.
Roster name casing changes
Mitigation: Slug derived case-insensitively. Cache key is the slug, so casing changes do not invalidate data.
Cost spike from naive queries
Mitigation: Date filters everywhere, 30-min cache, hard maximumBytesBilled = 1 GB so runaway queries fail fast.
